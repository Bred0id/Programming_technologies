# Описание задания

В данном задании требуется реализовать свою версию dict'а в Python: создайте пакет fastdict с классом FastDict внутри.

0. Сделайте ветку pybind в Вашем репозитории. В ней организуйте структуру:

pybind/
    test_time_memory.py
    test_results.md
    fastdict/
        CMakeLists.txt
        __init__.py
        (C++ исходники)
    build.sh (или аналогичный скрипт сборки)

1. Выберите структуру данных:

• Splay-дерево в репозитории https://github.com/jetminded/pybind_examples;
• Splay-дерево, реализованное Вами ранее;
• любое другое дерево (AVL, Red-Black и др.) или хеш-таблицу.

Если выбираете не Splay-дерево — согласуйте с преподавателем/ассистентом.

При доработке структуры данных, реализованной Вами в рамках курсов Программирование на C++/Алгоритмы и Структуры Данных возможен +1 балл.

!! Важно - использовать готовые STL контейнеры и биндить их (например, UnorderedMap) НЕЛЬЗЯ. Использовать готовый контейнер (в частности, std::vector) внутри вашего кода, реализующего другую структуру (например, хеш-таблицу) МОЖНО.

2. Доработайте так, чтобы узлы хранили ключи и значения; реализуйте код через шаблоны.

3. Добавьте весь необходимый функционал, чтобы Splay-дерево (теперь уже FastDict) имело интерфейс словаря (collections.abc.MutableMapping):

• __getitem__
• __setitem__
• __delitem__
• __iter__
• __len__
• __contains__
• keys()
• items()
• values()

Требования к объектам:

• значения: любые Python-объекты
• ключи: в дереве: должен быть определён __lt__, в хеш-таблице: должен быть определён __hash__

4. Доработайте сборку пакета:

• Сделайте биндинги для Python3 через pybind11;
• напишите CMakeLists.txt для сборки .so файла для Вашего кода;
• напишите скрипт build.sh, который запускает сборку и сохраняет so файл в корне папки fastdict.

Итоговый модуль должен импортироваться как from fastdict import FastDict

5. Напишите __init__.py для вашего пакета.

6. Напишите файл test_time_memory.py, в котором вы импортируете пакет fastdict и сравниваете затраты по времени и памяти FastDict и dict на различных операциях словарей (вставка, поиск, удаление данных). Попробуй подобрать набор данных и операций, на которых твоя реализация будет быстрее, чем встроенный dict. Проанализируй, почему так выходит - попробуй объяснить, в чём твой код оптимален, в чём - нет. Результаты экспериментов запиши в файл test_results.md.

P.S. В данном задании нет автопроверки. На репозитории https://github.com/jetminded/pybind_examples есть тесты, которые ваш код должен проходить.
