# Задание по git

> Технологии программирования, 2024г

---

## Условия задач

### Часть 1.

Слейте ветку `2` в текущую ветку (`1`)

При сливании возникнет конфликт. Разрешите его: в одном файле оставьте изменения из обоих веток, в итоговой версии файла числа должны идти по убыванию; другой же, который был удален на одной из веток, удалите.

Мерж-коммит должен иметь название `3`.

### Часть 2.

Вам снова предстоит разрешать мерж-конфликты. Итак, задание следующее: перебазируйте ветку `4` на текущую ветку (`3`). 

Для каждого из файлов оставьте изменения только из той ветки, чье название совпадает с названием файла.

### Часть 3.

Завершаем работу с мерж-конфликтами. В предыдущих двух частях мы учились разрешать мерж-конфликты, полученные в различных условиях, однако не все варианты были покрыты. В этой части вы должны будете создать мерж-конфликт, отличный ото всех, встреченных ранее.

Используя дополнительную ветку `6`, сделайте так, чтобы при после вызова на текущей ветке (`5`)

```shell
git merge 6
```

произошел мерж-конфликт.

### Часть 4.

Вернемся к ребейзу. Пусть ветки `8` и `9` независимы, т.е. отвечают за внесение изменений в различные участки кода (с несхожей функциональностью). Причем имеется следующая череда зависимостей веток: `7 -> 8 -> 9`.

 Ваша задача будет "сбалансировать это дерево", а именно перенести ветку `9` на ветку `7`.

### Часть 5.

Теперь будем тренировать исправление ошибок. Начнем с простого: вам необходимо отменить изменения, произведенные в коммите `21`. Новый коммит с изменениями назовите `Revert "21"`.

> Используйте предназначенную для этого команду. Прочие способы решения **не принимаются**!

### Часть 6.

Теперь же надо откатиться до коммита `24`.

### Часть 7.

Последним, что мы затронем в этой таске, будет исправление названий коммитов. В этой части вам достается более простой случай: исправьте имя последнего коммита (`28`) на `27`.

### Часть 8.

Усложним прошлую задачу: теперь вам необходимо отыскать коммит и изменить его имя.

1. Не пытайтесь влоб перебирать все коммиты, их почти 7000 - используйте механизмы, предоставляемые гитом

2. Нужный коммит - тот, в котором было добавлено ваше растение, последнее можно найти [здесь](https://docs.google.com/spreadsheets/d/1q3DZKI0-V4cJ29iZMQ0eaYRFRwTHqKw9xGfYgq9caI4/edit?usp=sharing). Также гарантируется, что растения добавлялись в лексикографическом порядке латинского языка.

3. Новое имя коммита должно быть `magnolia`

4. Не забудьте изменить файл `credentials.config`: вместо `[id]` вставьте свой *id* из таблицы


> При изменении файла `credentials.config` создайте коммит с названием `7000`, после чего сделайте **squash** с последним коммитом ветки.


### Часть 9.

Так как веток всего получилось 13, то придется добавить еще одну задачу)

Будем знакомиться с еще одной возможностью гита: переносить изменения выбранного коммита с одной ветки на другую. Итак, перенесите изменения из коммита `6003` ветки `15` на текущую ветку (`14`). В результате в файле `9.txt` числа должны идти по убыванию.

> При перенесении изменений сохраните имя коммита!
